《文字地狱大战》基础机关效果说明

一、基础交互物体

1. 地面

字符：地  
功能：行走平台  
玩家碰撞：✓ 可站立行走  
子弹碰撞：✓ 子弹击中时消失  
效果说明：
• 基础地面，角色可在其上行走、跳跃

• 阻挡子弹

• 不可破坏，永久存在

• 无特殊互动效果

2. 墙壁

字符：墙  
功能：障碍物  
玩家碰撞：✓ 无法通过  
子弹碰撞：✓ 子弹击中时消失  
效果说明：
• 垂直障碍



4. 陷阱

字符：火  
功能：接触伤害  
玩家碰撞：✗ 不阻挡移动，但接触造成伤害  
子弹碰撞：✗ 子弹可穿过  
效果说明：
• 接触时立即造成1点伤害

• 持续接触时每0.5秒造成1点伤害

• 不阻挡角色移动，可走过/穿过

• 视觉：火焰动画，有燃烧音效

5. 终点

字符：终  
功能：通关目标  
玩家碰撞：✗ 不阻挡，接触后过关  
子弹碰撞：✗ 子弹可穿过  
效果说明：
• 角色接触后立即通关当前关卡

• 接触时播放胜利特效和音效

• 子弹穿过终点无任何效果

• 通常放置在关卡最右端或谜题解决后位置

• 视觉：金色光芒闪烁效果

6. 洞

字符：洞  
功能：障碍  
玩家碰撞：✓ 无法通过，相当于障碍
子弹碰撞：✗ 子弹可穿过  
效果说明：
• 子弹可安全穿过空洞


二、机关系统

1. 门与开关系统

门（颜色前缀+门）

示例：红门、绿门、蓝门、黄门、紫门、青门  
功能：可开关的障碍门  
效果说明：
• 关闭状态：实体障碍，完全阻挡角色和子弹

• 开启状态：变为浅色，可通过

• 锁定状态：特殊状态，需额外条件解锁

• 颜色对应：只与同色开关联动

• 开关门时：有滑动动画和机械音效

开关（颜色前缀+开）

示例：红开、绿开、蓝开、黄开、紫开、青开  
功能：控制同色门状态  
效果说明：
• 攻击触发：被任何攻击命中时切换状态

• 状态切换：每次攻击切换一次（开↔关）

• 音效反馈：点击声

• 特殊：多个同色开关控制同一扇门时，需全部激活门才开

联动规则：
• 红开↔红门

• 绿开↔绿门

• 蓝开↔蓝门

• 黄开↔黄门

• 紫开↔紫门

• 青开↔青门

• 严格同色配对，不同颜色不联动

2. 传送门系统

传送起点

字符：传（前加颜色）  
示例：传红、传绿、传蓝、传黄、传紫、传青  
功能：传送起点，触碰后传送到对应终点  
效果说明：
• 触碰瞬间传送，有0.1秒延迟

• 传送动画：角色被吸入漩涡

• 传送音效：空间扭曲声

• 冷却：传送后1秒内免疫再次传送

• 视觉：入口处有漩涡旋转动画

传送终点

字符：送（前加颜色）  
示例：送红、送绿、送蓝、送黄、送紫、送青  
功能：传送终点，接收对应起点的传送  
效果说明：
• 接收传送，角色从此出现

• 出现动画：角色从漩涡中弹出

• 保护：出现后0.5秒无敌时间

• 视觉：稳定发光，有空间涟漪特效

配对与交换规则：
1. 基础配对：同色传与送自动配对
   • 传红↔送红

   • 传绿↔送绿

   • 等等

2. 位置交换：
   • 攻击同色开关可交换配对门的传送终点

   • 示例：攻击红开，传红的终点在送红A和送红B之间切换

   • 交换时：两传送终点闪烁提示

   • 应用：动态改变传送路径的解谜

3. 弓箭系统

弓箭发射器

字符：弓（可朝向不同方向）  

功能：可朝四个方向发射箭矢  
效果说明：
• 方向：固定朝一个方向发射

• 触发：被任何攻击命中时发射一支箭

• 只能发送一支箭

• 音效：弓弦声+发射声

弓箭子弹

字符：箭  
功能：弓箭发射的子弹  
效果说明：
• 飞行：直线飞行，直到碰撞

• 速度：中等飞行速度

• 伤害：命中敌人造成2点伤害

• 机关触发：可触发一次开关或机关

• 限制：每支箭只能触发一次机关，之后消失

• 视觉：箭矢飞行轨迹

• 音效：飞行嗖嗖声

特殊规则：
1. 箭命中非机关物体（墙、地等）时直接消失
2. 箭命中机关后，无论是否触发成功，该箭矢都消失


4. 灰尘系统

灰尘源

字符：脏  
功能：可被多次击打，生成灰尘  
效果说明：
• 耐久：最多承受5次击打

• 击打效果：每次击打生成1个"灰"字灰尘

• 视觉：被击打时变浅，直到变成白色消失，每次击打出的“灰”颜色跟“脏”的颜色一样

• 音效：击打沉闷声

坠落灰尘

字符：灰  
功能：击打"脏"后向下坠落的灰尘  
效果说明：
• 生成：从"脏"的位置向下坠落

• 重力：受重力加速下落

• 攻击效果：具有子弹攻击效果

  • 命中敌人：造成1点伤害

  • 命中开关：触发机关一次

  • 命中其他：无效果

• 消失：落地后立即消失

• 视觉：灰尘颗粒特效

• 音效：轻微撞击声

数量限制：
• 每个"脏"生成5个"灰"



应用示例：

脏（耐久5）
↓ 攻击
灰 生成并下落
↓ 命中开关
红开 被触发
红门 状态切换

四、基础谜题示例

示例1：简单开门谜题


场景：
[墙][墙][墙][墙][墙]
[墙]         [红门]
[墙]   [红开]     [墙]
[墙][墙][墙][墙][墙]

解法：
1. 攻击红开
2. 红门开启
3. 通过红门


示例2：弓箭触发谜题


场景：
[墙][墙][墙][墙][墙][墙]
[墙]       [弓→]   [绿门]
[墙]                 [墙]
[墙]   [绿开]         [墙]
[墙][墙][墙][墙][墙][墙]

解法：
1. 攻击弓→
2. 弓发射箭→
3. 箭命中绿开
4. 绿门开启
5. 通过绿门


示例3：灰尘系统谜题


场景：
[墙][墙][墙][墙][墙]
[墙]   [脏]   [蓝门]
[墙]             [墙]
[墙]     [蓝开]   [墙]
[墙][墙][墙][墙][墙]

解法：
1. 攻击脏（生成灰）
2. 灰下坠落中命中蓝开
3. 蓝门开启
4. 通过蓝门
注意：需站在合适位置让灰能落到开关上


示例4：传送门交换谜题


场景：
[墙][墙][墙][墙][墙][墙]
[墙][传红A]     [送红A]
[墙]             [红开]
[墙][传红B]     [送红B]
[墙][墙][墙][墙][墙][墙]

初始：传红A→送红A，传红B→送红B
解法：
1. 进入传红A到达送红A（位置1）
2. 攻击红开
3. 传送配对交换：传红A→送红B，传红B→送红A
4. 进入传红A到达送红B（位置2）


此设计确保：
1. 每个部件功能单一明确
2. 交互关系清晰可预测
3. 组合时能产生有趣谜题
4. 符合直觉，易于理解
5. 为复杂谜题设计提供基础组件
